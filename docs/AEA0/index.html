1. Què és Vanilla Javascript?

El terme vanilla javascript s'utilitza per referirse a javascript sense cap tipus
de llibreria o extensió.

2. Diferència els ECMA Script que pertanyen a Javascript i a Javascript Modern.

Sempre es considerarà 'Modern Javascript' l'ultima generació de frameworks i
llibreries. Ara mateix es pot considerar Deno pero fa 7 anys va ser React.

3. Qui va crear Javascript? De quina empresa era?

Brendan Eich, que treballava a Netscape, va ser asignat crear un nou llenguatge
d'scripting. El seu nom en codi va ser 'Mocha' pero després es va passar a dir
LiveScript i no molt més tard va acabar dient-se Javascript per aprofitar-se de
la fama que en aquell moment tenia Java.

4. Quina empresa hi ha actualment darrera de Javascript?

Darrera del estandar ECMAScript (de la que Javascript depen) hi ha la ECMA
(European Computer Manufacturers Association)

5. Què és una arrow function? A quin ES apareix (quin any).

Una arrow funcion es una forma compacta i senzilla de utilitzar funcions.
Apareix en el ES6 al any 2015

6. Què és un callback? Posa un exemple.

A Javascript podem passar funcions com parametres per altres funcions. Això es
anomenat una funció callback
En aquest exemple utilitzarem setTimeout que es una funció basica de JS i com a
argument li donarem una altra funció:
function hola() {
console.log("Hola");
}
setTimeout(hola(), 3000);

7. Què és una promesa en Javascript (promise). Posa un exemple.
Explica també la diferència entre la programació síncrona i la programació asíncrona, posa un exemple.

Amb una promesa deixes a javascript esperant a algo mentre segueix executant codi
Pot tenir tres estats: pendent, completat o fallit. Un exemple pot ser:
let promesa = new Promise(function(resolve, reject) {
resolve("No errors!");
}
Podem completar o fallar una promesa amb resolve i reject.
